라이브 점검 요약
- https://88st.cloud/analysis/ 는 403 Forbidden 응답
- https://88st.cloud/odds/ 는 정상 200 응답
- 정적 파일에는 analysis/index.html 이 존재하므로, 실서비스 403은 누락보다 엣지/보안 규칙 영향 가능성이 큼

가장 가능성 높은 원인
1) Cloudflare Access 또는 WAF 규칙이 /analysis* 경로를 차단
2) 별도 Worker/Route 규칙이 /analysis 경로를 가로채서 403 반환
3) Pages 프로젝트가 /analysis 경로만 보호 대상으로 남아 있음

이번 패치의 즉시 완화 조치
- 메인 스포츠 위젯의 /analysis 링크를 /odds/ 로 교체
- _redirects 에 /analysis -> /odds/ 규칙 추가
- analysis/index.html 자체도 /odds/ 로 리다이렉트 안내 추가
- 모바일 독 분석 링크도 /odds/ 로 교체

Cloudflare에서 직접 확인할 항목
- Access > Applications 에 /analysis* 보호 규칙 존재 여부
- Security > WAF > Custom rules 에 /analysis 차단 규칙 존재 여부
- Workers & Routes 에 /analysis* 라우트가 걸려 있는지
- Pages 프로젝트의 Functions / Redirects / Headers 에 경로별 차단이 있는지

권장
- 당장은 사용자 동선을 /odds/ 로 통일
- Cloudflare 쪽 규칙 제거 후 /analysis 재오픈 여부 결정
