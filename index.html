검증 · 조회 · 전환 흐름을
첫 화면에서 바로 실행할 수 있게 정리했습니다.

기존 Worker, 주소, 배당분석, 광고 구조는 유지하고 사례 조회 · 도메인 점검 · 브랜드 결과 · 결과 공유만 더 빠르게 쓰이도록 재정렬했습니다. 모바일에서도 첫 화면에서 바로 실행할 수 있게 CTA 흐름을 짧게 잡았습니다.

먹튀 사례 조회도메인 검사브랜드 결과 모음배당분석 열기
즉시 실행 01먹튀 사례 조회구글 검색어 6종을 바로 생성해 브랜드명·도메인 흔적부터 확인즉시 실행 02도메인 검사RDAP, DNS, IP, ASN 기준으로 등록 이력과 대역 힌트 점검즉시 실행 03브랜드 결과판광고카드 4종과 연결되는 정적 결과 페이지 모음 바로 열기즉시 실행 04공유 결과 생성상담·운영에 재사용할 링크형 결과 페이지 바로 생성
광고카드 고급화광고카드마다 공식 도메인·가입코드·브랜드 결과 버튼을 함께 배치해 전환 흐름을 분리했습니다.
브랜드 결과 구조양심, 칠벳, 베가스, 어벤저스벳 4종을 정적 결과판 구조로 묶어 재사용 가능한 랜딩 자산으로 만들었습니다.
모바일 CTA 밀도 강화첫 화면에서 검색·검사·결과판·공유를 한 번에 눌러볼 수 있게 CTA 밀도를 재조정했습니다.

핵심 목표

검색 유입 → 사례 조회 → 도메인 검사 → 브랜드 결과 확인 → 광고카드 전환 흐름을 메인에서 바로 이어주도록 설계했습니다.

브랜드 결과판

광고 운영 중인 4개 브랜드를 정적 결과 구조로 만들어 공식 주소, 코드 복사, 검색 포인트를 고정 경로로 관리합니다.

운영 기준 요약

메이저 판단과 광고 전환을 동시에 가져가기 위해 메인에서 먼저 보여줄 핵심 기준만 남겼습니다.

검색 흔적 확인브랜드명과 도메인을 함께 검색해 같은 문제 문구가 반복되는지 먼저 확인합니다. 검색은 빠르고 실제 후기 축적을 보기에 좋습니다.
도메인 이력 점검신규 도메인, 잦은 주소 변경, 만료 임박, 불안정한 네임서버는 별도 주의 신호로 분리해 확인하는 편이 안전합니다.
소액 테스트 원칙검증 도구 결과만으로 끝내지 말고, 입금·출금·고객센터 응답까지 소액 기준으로 실제 동작을 확인해야 합니다.

리뉴얼 확장 완료

광고카드 고급화 + CTA 재배치 + 브랜드별 정적 결과 구조를 붙였습니다.

이번 단계부터는 메인 첫 화면 자체가 독립 운영형 진입판 역할을 하며, 브랜드가 늘어나도 동일한 템플릿으로 정적 결과 페이지를 자동 생성하는 방향으로 확장할 수 있습니다.

브랜드가 추가되면 assets/data/brand.results.v1.20260330.json을 수정하고 node scripts/generate-brand-pages.mjs를 다시 실행하면 같은 구조의 정적 결과 페이지를 계속 늘릴 수 있습니다.
