검증 중심 독립 운영 모드로 재정렬했습니다.

이번 리뉴얼은 기존 88st.cloud 프로젝트를 버리지 않고, Worker·기존 API 사용부·광고카드 구조·배당분석 기능은 유지하면서도 실제 유입과 전환에 필요한 먹튀 사례 조회 · 도메인 검사 · 결과 페이지 공유 · 상담 동선만 앞으로 나오도록 메인을 재구성한 것입니다. 불필요한 게시글형 진입 구조는 정리하고, 핵심 검증 기능을 모바일에서도 바로 쓰게 바꿨습니다.

안전센터 보기먹튀 사례 조회배당분석 열기
기존 기능 유지배당분석, 광고카드, Worker API, 관리자/운영 페이지는 그대로 유지한 상태에서 구조만 정리했습니다.
검증 동선 집중사용자가 메인에서 바로 사례 검색과 도메인 검사를 시작하고 결과 링크까지 만들 수 있게 진입 단계를 짧게 줄였습니다.
모바일 우선 재배치카테고리 카드와 긴 글 목록 대신 눌러야 하는 기능 버튼과 전환 요소를 위쪽으로 끌어올렸습니다.

핵심 목표

검색 유입 → 사례 조회 → 도메인 검사 → 결과 링크 공유 → 광고카드 전환 흐름을 메인에서 바로 이어주도록 설계했습니다.

운영 기준 요약

메이저 판단과 광고 전환을 동시에 가져가기 위해 메인에서 먼저 보여줄 핵심 기준만 남겼습니다.

검색 흔적 확인브랜드명과 도메인을 함께 검색해 같은 문제 문구가 반복되는지 먼저 확인합니다. 검색은 빠르고 실제 후기 축적을 보기에 좋습니다.
도메인 이력 점검신규 도메인, 잦은 주소 변경, 만료 임박, 불안정한 네임서버는 별도 주의 신호로 분리해 확인하는 편이 안전합니다.
소액 테스트 원칙검증 도구 결과만으로 끝내지 말고, 입금·출금·고객센터 응답까지 소액 기준으로 실제 동작을 확인해야 합니다.

카테고리형/블로그형 메인 구조를 걷어냈습니다.

이번 단계에서는 기존 게시글 묶음과 카테고리 진입 구조 대신, 실제 운영에 필요한 검증 허브 중심 구조만 전면 배치했습니다. 이후 확장은 이 구조 위에 기능형 페이지를 추가하는 방식이 가장 안전합니다.

GitHub/Cloudflare Pages에서는 전체 ZIP 기준 덮어쓰기 배포가 가장 안전합니다. 부분 패치 ZIP은 수정 파일 확인용으로만 쓰고, 삭제 반영까지 하려면 전체 ZIP으로 교체하는 편이 맞습니다.
